Thank you for your interest in contributing to PLACE@ASET! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Issues

1. Check if the issue already exists in the issue tracker.
2. Create a new issue with a clear title and detailed description.
3. Include steps to reproduce (for bugs) or detailed requirements (for features).
4. Add relevant labels (bug, feature, enhancement, etc.).

### Submitting Code

1. **Fork** the repository and create a feature branch from `main`.
2. **Follow** the coding standards and folder structure.
3. **Write** tests for new functionality.
4. **Document** any new APIs or features.
5. **Submit** a pull request with a clear description of changes.

### Code Standards

- **JavaScript**: Follow ESLint configuration.
- **CSS**: Use vanilla CSS with the design system variables.
- **Components**: Keep components focused and reusable.
- **API**: Follow the REST conventions and response format.
- **Commits**: Use clear, descriptive commit messages.

### Branch Naming

- `feature/description` — New features
- `fix/description` — Bug fixes
- `docs/description` — Documentation changes
- `refactor/description` — Code refactoring

## Code of Conduct

- Be respectful and inclusive.
- Provide constructive feedback.
- Focus on the technical merits of contributions.
- Help maintain a positive community environment.
